DHCP: Give Me My Network Settings

When you connect to WiFi at a coffee shop, your device automatically receives network settings without manual input. This happens using DHCP (Dynamic Host Configuration Protocol).

Why DHCP?

  • Automatically assigns IP address
  • Avoids IP conflicts
  • Saves manual configuration

DHCP Process (DORA)

  1. Discover – Client searches for DHCP server
  2. Offer – Server offers IP address
  3. Request – Client accepts the offer
  4. Acknowledge – Server confirms assignment

Key Answers

Steps: 4

Destination IP: 255.255.255.255

Source IP: 0.0.0.0

ARP: Mapping IP to MAC

ARP helps convert IP addresses into MAC addresses so devices can communicate on the same network.

Example:

  • Request MAC: ff:ff:ff:ff:ff:ff
  • MAC of 192.168.66.1: 44:df:65:d8:fe:6c

ICMP: Network Troubleshooting

Ping

Checks if a system is alive and measures response time.

Traceroute

Shows the path packets take using TTL (Time To Live).

Key Facts:

  • Ping uses ICMP Echo Request/Reply
  • Traceroute depends on TTL reaching zero

Routing

Routing determines the best path for data across networks.

Protocols:

  • OSPF
  • EIGRP
  • BGP
  • RIP

NAT (Network Address Translation)

NAT allows multiple devices to share one public IP.

Example:

  • Public IP: 212.3.4.5
  • Connections: ~65,000
